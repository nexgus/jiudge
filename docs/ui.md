# UI 模式與狀態機 - 路徑規劃

本文件定義路徑規劃功能的 UI 模式, 按鍵與模式轉換. 以使用者定義的三模式為主, 並補足實作必然要面對的缺口 (標示 "(補充)" 者為原定義未涵蓋, 由本文件補上, 可再調整).

術語: 本文件中, 地圖上由規劃產生的折線稱 "規劃路徑". 按鍵 "清除軌跡" 清除目前顯示於地圖的規劃路徑. (補充: Phase 1 加入 GPS 軌跡錄製後, "軌跡" 一詞會另有所指; 屆時此鍵語義可擴及已顯示的錄製軌跡. 建議規劃情境統一用 "路徑", 但此處保留指定鍵名 "清除軌跡".)

---

## 1. 模式總覽

| 模式 | 程式代號 (建議) | 動作列按鍵 |
|------|------|------|
| 地圖檢視模式 | `MAP_VIEW` | 清除軌跡, 規劃路徑 |
| 路徑編輯模式 | `ROUTE_EDIT` | +, -, 儲存, 取消 |
| 路徑檢視模式 | `ROUTE_VIEW` | 編輯, 離開 |

預設: App 啟動進入 **地圖檢視模式** (補充).

---

## 2. 各模式行為

### 2.1 地圖檢視模式 (MAP_VIEW)

兩個按鍵:

- **清除軌跡** -> 清除地圖上所有規劃路徑, 留在地圖檢視模式.
  - (補充) 僅在地圖上有顯示路徑時可用; 無路徑時停用.
- **規劃路徑** -> 彈出視窗, 三選一:
  - **新規劃** -> 清除所有路徑, 進入路徑編輯模式 (空白規劃).
    - (補充) 若 BRouter 路由資料未就緒, 不進入編輯, 改以訊息提示 (無路由資料無法計算路徑).
  - **載入已存檔** -> 清除所有路徑, 載入所選檔案, 進入 **路徑檢視模式**.
    - (補充) 列出已存路徑供選擇; 若無任何已存檔, 顯示 "尚無已儲存的路線".
  - **取消** -> 退回地圖檢視模式 (不變更).

### 2.2 路徑編輯模式 (ROUTE_EDIT)

畫面中央顯示十字 (補充: 標示新增經過點的落點 = 地圖中心). 四個按鍵:

- **+** -> 以十字處 (地圖中心) 新增一個經過點; 將上一個經過點與此經過點之間的路徑畫出, 多條可選時取最短路徑 (BRouter).
  - (補充) 若無法連到該點 (離路網太遠 / 無路徑 / 路由資料缺), 以訊息提示且不新增該點.
  - (補充) 計算期間動作列暫時停用並顯示進度指示.
- **-** -> 刪除最後一個經過點, 連同其前一段路徑.
  - (補充) 無經過點時停用.
- **儲存** -> 命名並儲存路徑, 進入路徑檢視模式 (顯示剛儲存的路徑).
  - (補充) 經過點少於 2 (尚無任何路段) 時停用.
  - (補充) 命名規則見 4.2.
- **取消** -> 放棄本次所有編輯, 還原為 "基準" 路徑, 進入路徑檢視模式.
  - (補充) "基準" 定義見 3; 若基準為空 (新規劃且未曾儲存即取消), 無可檢視內容, 改回地圖檢視模式 (空).

### 2.3 路徑檢視模式 (ROUTE_VIEW)

顯示一條路徑 (剛儲存的, 或載入的). 兩個按鍵:

- **編輯** -> 進入路徑編輯模式, 以目前顯示的路徑為可編輯內容.
  - (補充) 進入編輯時, 此路徑成為該次編輯的 "基準" (見 3).
- **離開** -> 不清除路徑, 回到地圖檢視模式 (路徑仍顯示於地圖, 可在地圖檢視模式用 "清除軌跡" 清除).

---

## 3. "基準" 路徑與 "取消" 的還原語義 (補充)

路徑編輯模式的 "取消" 需要一個還原目標, 稱為 **基準**:

- 由 **路徑檢視模式 -> 編輯** 進入編輯: 基準 = 進入前所檢視的路徑.
- 由 **新規劃** 進入編輯: 基準 = 空.
- **儲存** 成功後: 剛儲存的路徑成為新的基準.

"取消" = 放棄本次編輯的增刪, 還原為基準:

- 基準非空 -> 進入路徑檢視模式, 顯示基準.
- 基準為空 (新規劃從未儲存) -> 回地圖檢視模式 (不顯示任何路徑).

例: 載入舊檔 (檢視) -> 編輯 -> 加兩點 -> 取消, 會還原成載入時的原樣並回到檢視模式; 而非清空.

## 4. 轉換表與規則

### 4.1 模式轉換表

| 來源模式 | 動作 | 目標模式 | 對地圖路徑的影響 |
|------|------|------|------|
| 地圖檢視 | 清除軌跡 | 地圖檢視 | 清除目前顯示的路徑 |
| 地圖檢視 | 規劃路徑 -> 新規劃 | 路徑編輯 | 清空, 開新的空白規劃 (基準=空) |
| 地圖檢視 | 規劃路徑 -> 載入已存檔 | 路徑檢視 | 清空, 載入所選檔並顯示 (基準=該檔) |
| 地圖檢視 | 規劃路徑 -> 取消 | 地圖檢視 | 不變 |
| 路徑編輯 | + | 路徑編輯 | 新增經過點並畫出最短路徑 |
| 路徑編輯 | - | 路徑編輯 | 刪除最後一個經過點與其路段 |
| 路徑編輯 | 儲存 | 路徑檢視 | 命名存檔; 顯示剛存路徑 (成為新基準) |
| 路徑編輯 | 取消 | 路徑檢視 / 地圖檢視 | 還原為基準; 基準為空則回地圖檢視 |
| 路徑檢視 | 編輯 | 路徑編輯 | 以目前路徑為可編輯內容 (成為基準) |
| 路徑檢視 | 離開 | 地圖檢視 | 保留路徑顯示, 不清除 |

### 4.2 命名與儲存 (補充)

- "儲存" 彈出命名輸入框; 留空則用預設名 (例: "未命名路線").
- 編輯既有路徑後再存時, 名稱欄預填原名.
- v1: 每次儲存產生一個新檔 (以建立時間戳區別), 不就地覆蓋. 就地覆蓋 / 改名列為 v2 候選.
- 儲存成功後以 snackbar 提示 (例: "已儲存規劃路徑: <名稱>").
- 首次儲存或載入時, 先跳系統資料夾選擇器讓使用者挑一個存放資料夾 (路徑存於該資料夾下的 `Jiudge/plans/`, 屬公共儲存, 撐過解除安裝). 選定後會記住, 之後不再詢問; 若使用者在選擇器按取消, 該次儲存/載入即中止.

### 4.3 縮放與地圖控制 (補充, 待確認位置)

- 縮放 +/- 與層級讀數為地圖通用控制, 三個模式皆持續顯示, 與各模式動作列獨立.
- 版面: 動作列置於畫面下方; 縮放控制置於右側 (編輯模式動作列佔底部整列, 縮放需避開, 建議右上或右中).

### 4.4 旋轉與橫向 (補充)

依 spec 1.2, 地圖類畫面支援直 / 橫向自適應; 各模式動作列隨之重新排版.

## 5. 狀態圖

```
                         啟動
                          |
                          v
                 +-------------------+
   清除軌跡 ----> |   地圖檢視模式     | <----------- 離開 (保留路徑)
   (清除路徑)     |  [清除軌跡][規劃]  | <--- 規劃路徑->取消         \
                 +-------------------+                            \
                    |                                              \
          規劃路徑 -> 新規劃 (基準=空)        規劃路徑 -> 載入 (基準=該檔) \
                    v                                   v             |
            +---------------+   儲存 / 取消      +---------------+      |
            |  路徑編輯模式  | ----------------> |  路徑檢視模式  | -----+
            |  [+][-]       |                   |  [編輯][離開]  |
            |  [儲存][取消]  | <---------------- |               |
            +---------------+      編輯          +---------------+
              (基準為空且取消 -> 回地圖檢視模式)
```

## 6. 與現行實作的落差 (供後續調整, 本文件不含程式變更)

目前程式 (本次 session 已實作的版本) 與上述定義的差異:

1. 地圖檢視模式尚無 "清除軌跡" 鍵 (需新增; 含 "有路徑才可用" 的啟用條件).
2. "規劃路徑" 彈窗目前靠點視窗外關閉, 無明確 "取消" 鍵 (需補).
3. 載入已存檔目前進入 **編輯模式**, 應改為進入 **路徑檢視模式**.
4. 按鍵標籤需統一為中文: 編輯模式 "儲存" / "取消"; 檢視模式 "編輯" / "離開"; 彈窗 "新規劃" / "載入已存檔" / "取消". ("+" / "-" 維持符號.)
5. 編輯模式 "取消" 目前為清除並回主畫面, 應改為 **還原基準並進檢視模式** (基準為空才回地圖檢視).
6. 需引入 "基準路徑" 快照機制 (見 3).
7. 檢視模式 "離開" 需確保 **保留** 地圖上的路徑 (目前 "關閉檢視" 為清除).
8. 新規劃前的路由資料就緒檢查維持現狀.
